# 测试模式说明

## 功能概述

在配置文件中添加了 `test_mode` 参数，当设置为 `1` 时，程序会每隔1秒自动调用 `handle_vehicle_status` 函数，模拟车辆状态数据上传。

## 配置方法

在 `config.json` 文件中添加 `test_mode` 参数：

```json
{
  "vehicle_id": "V001",
  "mqtt": {
    "broker": "localhost",
    "port": 1883,
    "username": "lx",
    "password": "lxtest"
  },
  "websocket": {
    "host": "0.0.0.0",
    "port": 8765
  },
  "command_mapping_file": "command_mapping.json",
  "test_mode": 1
}
```

### 参数说明

- `test_mode`: 
  - `0` (默认): 关闭测试模式，正常接收来自车端程序的ROS2状态数据
  - `1`: 启用测试模式，每隔1秒自动生成模拟数据并调用 `handle_vehicle_status`

## 模拟数据说明

测试模式会生成以下模拟数据：

### 车辆位置
- `cur_x`: 在基础位置(100.0)附近随机变化，范围 ±5.0
- `cur_y`: 在基础位置(200.0)附近随机变化，范围 ±5.0
- `rtk_flag`: 固定为 1（RTK定位标志）

### 电池电量
- `battery`: 从85.0%开始，每次减少0.1%，最低降至20.0%

### 速度
- `v`: 在1.5-3.5 m/s之间随机变化

### 工作模式
- `driving_mode`: 随机选择 0（手动）或 1（自动）
- `clean_mode`: 随机选择 0（关闭）或 1（开启）
- `water_mode`: 随机选择 0（关闭）或 1（开启）

## 数据格式

模拟数据会按照以下格式生成：

```json
{
  "type": "vehicle",
  "data": {
    "cur_x": 102.35,
    "cur_y": 198.67,
    "rtk_flag": 1,
    "battery": 84.9,
    "v": 2.8,
    "driving_mode": 1,
    "clean_mode": 1,
    "water_mode": 0
  },
  "timestamp": "2026-01-11T14:30:00.123456"
}
```

## 使用方法

### 1. 启用测试模式

编辑 `config.json`，将 `test_mode` 设置为 `1`：

```json
{
  "test_mode": 1
}
```

### 2. 启动程序

```bash
python3 communication_bridge.py
```

### 3. 查看日志

程序启动后，会看到以下日志：

```
测试模式已启用：每隔1秒模拟数据上传
测试模式：开始模拟数据上传
```

在DEBUG级别下，会看到每次上传的数据详情：

```
测试模式：模拟数据已上传 - 位置(102.35, 198.67), 电池84.9%, 速度2.8m/s
```

### 4. 数据流向

启用测试模式后，模拟数据会：

1. 调用 `handle_vehicle_status()` 函数
2. 转发给WebSocket客户端（如果有连接）
3. 转发给MQTT（发布到 `/vehicle2cloud/{vehicle_id}/vehicleinfo`）

## 注意事项

1. **测试模式与正常模式互斥**：
   - 启用测试模式时，程序仍会接收ROS2状态数据（如果车端程序在运行）
   - 建议在测试时关闭车端程序，避免数据冲突

2. **数据变化**：
   - 位置数据会在基础位置附近随机变化
   - 电池电量会持续下降（模拟真实场景）
   - 速度和工作模式会随机变化

3. **性能影响**：
   - 测试模式会创建一个独立的线程
   - 对程序性能影响很小
   - 可以通过设置 `test_mode: 0` 随时关闭

4. **日志级别**：
   - 测试模式启动信息：INFO级别
   - 每次上传的数据详情：DEBUG级别
   - 如需查看详细数据，请将日志级别设置为DEBUG

## 关闭测试模式

将 `config.json` 中的 `test_mode` 设置为 `0` 或删除该参数，然后重启程序：

```json
{
  "test_mode": 0
}
```

或者直接删除 `test_mode` 参数（默认为0）。

## 应用场景

1. **开发测试**：在没有实际车端程序时，测试前端和云平台的数据接收功能
2. **功能验证**：验证WebSocket和MQTT的数据转发功能
3. **性能测试**：测试系统在高频数据上传下的性能
4. **演示展示**：在演示环境中展示实时数据更新

## 故障排查

### 问题1：测试模式未启动

**检查项**：
- 确认 `config.json` 中 `test_mode` 设置为 `1`
- 查看日志是否有 "测试模式已启用" 的提示
- 确认程序已正常启动

### 问题2：数据未上传

**检查项**：
- 查看日志是否有错误信息
- 确认 `handle_vehicle_status` 函数正常工作
- 检查WebSocket和MQTT连接状态

### 问题3：数据格式错误

**检查项**：
- 确认数据格式符合 `handle_vehicle_status` 的要求
- 查看日志中的错误信息
- 检查数据字段是否完整
